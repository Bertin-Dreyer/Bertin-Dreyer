## About

I'm a full-stack developer building toward a career in fintech and enterprise software — the kind of systems that move money, manage wealth, and operate under real pressure.

My foundation is a **BSc in Computing from UNISA**, which gave me depth in data structures, database design, algorithms, and systems architecture. I paired that with Angela Yu's Full Stack Web Development Bootcamp and a self-directed curriculum I built around the PERN stack and financial domain problems. Structured environments have never been where I learn best — I do my best work when I own the problem and design the path.

I'm drawn to fintech specifically because I like business and I like pressure. Financial software has to be correct. It has to be auditable. The margin for ambiguity is zero, and that's exactly the kind of problem space I want to work in.

My flagship project, **OMNIPAY-Legacy**, is a personal tribute to my late father — a developer and DBA who built systems for Old Mutual's OmniPay division, which processed government pension distributions across South Africa. I'm rebuilding that world from scratch: double-entry ledger, three-schema PostgreSQL, POPIA/FICA/SARB compliance, cent-precision arithmetic. Not because it's impressive on a CV — because it matters to me.

I'm ambitious beyond what people expect from a junior with no formal experience yet. That gap is closing fast.

### ExchangeAPI
*Currency exchange REST API with a production-minded architecture*

A lightweight currency conversion service built around real fintech constraints. Implements a pre-fetch middleware layer to eliminate redundant external API calls, triangulation formula to convert between any two of 160+ currencies using USD as a bridge, NaN defensive validation, and floating-point precision handling. Includes an interactive web UI for live conversion.

**Stack:** Node.js · Express 5 · REST APIs · MVC Architecture

[![Repo](https://img.shields.io/badge/Repo-View-181717?style=for-the-badge&logo=github&logoColor=white)](https://github.com/Bertin-Dreyer/ExchangeAPI)

---

### Personal Budget Tracker
*A deployed fintech-patterned expense tracker with production-grade security*

A full-stack PERN application and the project where I went deep on financial API design. Implements JWT authentication with refresh token rotation, idempotency keys to prevent duplicate transactions on network retries, BOLA protection on every data endpoint, BIGINT cent-precision storage, and a transaction status lifecycle (PENDING / COMPLETED / FAILED). Deployed on Vercel and Railway.

**Stack:** PostgreSQL · Express · React 18 · Vite · JWT · bcrypt · Helmet.js · express-rate-limit · Axios · express-validator

[![Repo](https://img.shields.io/badge/Repo-View-181717?style=for-the-badge&logo=github&logoColor=white)](https://github.com/Bertin-Dreyer/PersonalBudgetTracker)
[![Live Demo](https://img.shields.io/badge/Live-Demo-00d4aa?style=for-the-badge)](https://wallet-dashboard-azure.vercel.app)

### OMNIPAY-Legacy · Omni Wallet
*A South African digital wallet built to honour the original OmniPay system*

A full-stack PERN application with a double-entry ledger, three-schema PostgreSQL architecture (identity / financial / audit), BIGINT cent-precision storage, and database-trigger audit logs. Designed for POPIA, FICA, and SARB regulatory compliance. Named after the system my late father built as a developer and DBA at Old Mutual — a platform that processed government pension distributions across South Africa.

**Stack:** PostgreSQL 15 · Express 5 · React · Node 24 · JWT

[![Repo](https://img.shields.io/badge/Repo-View-181717?style=for-the-badge&logo=github&logoColor=white)](https://github.com/Bertin-Dreyer/omni-wallet)
